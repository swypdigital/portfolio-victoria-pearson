## VERSION CONTROL PROTOCOL

### File Naming Convention:
```
GENCRAFT_V3_MASTER_REBUILD_PLAN_v<SEMANTIC>_<DATE>_<TIMESTAMP>_<CHANGE_TYPE>.md
```

**Components:**
- `<SEMANTIC>`: Semantic version (v3.0.0, v3.0.1, v3.1.0, etc.)
- `<DATE>`: YYYY-MM-DD format
- `<TIMESTAMP>`: HHMMSS format (24-hour)
- `<CHANGE_TYPE>`: One of:
  - `BASELINE` - Original version / major milestone
  - `MODIFIED` - Edits to existing content
  - `ADDED` - New sections added
  - `CORRECTED` - Bug fixes / error corrections
  - `EXPANDED` - Additional detail added
  - `CLARIFIED` - Clarifications / notes added
  - `RESTRUCTURED` - Organizational changes

### Semantic Versioning Rules:
- **Major (X.0.0)**: Complete plan overhaul, architecture changes
- **Minor (3.X.0)**: New phases added, significant scope changes
- **Patch (3.0.X)**: Clarifications, corrections, minor additions

### Version Control Workflow:
1. **NEVER overwrite existing files**
2. **ALWAYS create dated/timestamped copy BEFORE editing**
3. **Update VERSION_HISTORY.md with changes**
4. **Commit to git with descriptive message**
5. **Keep "HEAD" file (GENCRAFT_V3_MASTER_REBUILD_PLAN.md) as current version**
